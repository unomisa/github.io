SQL

⁢⁢SQL是结构化查询语言的缩写，用来访问和操作数据库系统。

不同的数据库，都支持SQL。即可通过SQL语句操作不同的数据库(增删改查)。

关系数据库

  1. 关系数据库是建立在关系模型上的。而关系模型本质上就是若干个存储数据的二维表，可以把它们看作很多Excel表。

    • 表的每一行称为记录（Record），记录是一个逻辑意义上的数据。一条记录就是由多个字段组成的。

    • 表的每一列称为字段（Column），同一个表的每一行记录都拥有相同的若干字段。

  2. 关系数据库的表和表之间需要建立“一对多”，“多对一”和“一对一”的关系，这样才能够按照应用程序的逻辑来组织和存储数据。

  3. 在关系数据库中，关系是通过主键和外键来维护的。

    主键：
    ▍⁢关系表有个很重要的约束：能够通过某个字段唯一区分出不同的记录，这个字段被称为主键。

    ①插入相同主键的两条记录是不被允许的。
    ②记录一旦插入到表中，主键最好不要再修改，因为主键是用来唯一定位记录的，修改了主键，会造成一系列的影响。

    选取主键的一个基本原则是：不使用任何业务相关的字段作为主键。

    联合主键：
    ▍⁢关系数据库还允许通过多个字段唯一标识记录，即两个或更多的字段都设置为主键，这种主键被称为联合主键。
    ②对于联合主键，允许一列有重复，只要不是所有主键列都重复即可。

    没有必要的情况下，我们尽量不使用联合主键，因为它给关系表带来了复杂度的上升。

    外键：
    ▍⁢在表中，通过某个字段，可以把数据与另一张表关联起来，这种列称为外键。
    ①外键通过定义外键约束实现：

    例如：
    ALTER TABLE students
    ADD CONSTRAINT fk_class_id      //外键约束名称：任意
    FOREIGN KEY (class_id)          //指定字段作为外键
    REFERENCES classes (id);        //关联到classes表的id列（即classes表的主键）。

    ​ 通过定义外键约束，关系数据库可以保证无法插入无效的数据。
    ​ 即如果classes表不存在id=99的记录，students表就无法插入class_id=99的记录。

    外键约束会降低数据库的性能。

    ②删除一个外键约束

    例如：
    ALTER TABLE students   //表关于
    DROP FOREIGN KEY fk_class_id; //删除指定的外键名称

关系数据库支持的标准数据类型

名称 类型 说明
INT 整型 4字节整数类型，范围约+/-21亿
BIGINT 长整型 8字节整数类型，范围约+/-922亿亿
REAL(FLOAT) 浮点型 4字节浮点数，范围约+/-1038
DOUBLE 浮点型 8字节浮点数，范围约+/-10308
DECIMAL(M,N) 高精度小数 由用户指定精度的小数，例如，DECIMAL(20,10)表示一共20位，其中小数10位，通常用于财务计算
CHAR(N) 定长字符串 存储指定长度的字符串，例如，CHAR(100)总是存储100个字符的字符串
VARCHAR(N) 变长字符串 存储可变长度的字符串，例如，VARCHAR(100)可以存储0~100个字符的字符串
BOOLEAN 布尔类型 存储True或者False
DATE 日期类型 存储日期，例如，2018-06-22
TIME 时间类型 存储时间，例如，12:20:59
DATETIME 日期和时间类型 存储日期+时间，例如，2018-06-22 12:20:59

通常情况下，字段应该避免允许为NULL。不允许为NULL可以简化查询条件，加快查询速度，也利于应用程序读取数据后无需判断是否为NULL。

MySQL

  1. MySQL是目前应用最广泛的开源关系数据库。

  2. MySQL本身实际上只是一个SQL接口，它的内部包含了多种数据引擎。
    例如：

    • InnoDB：由Innobase Oy公司开发的一款支持事务的数据库引擎，2006年被Oracle收购；
    • MyISAM：MySQL早期集成的默认数据库引擎，不支持事务。

    切换MySQL引擎不影响写的应用程序使用MySQL的接口。
